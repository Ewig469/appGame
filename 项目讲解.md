# BrückenSpiel — 项目讲解

## 项目概述

实现双人策略棋盘游戏「Brücken」。目的是在两个对边之间用「桥梁」连接出一条通路。
本项目基于哥廷根大学 APP SoSe 2026 课程预设库 (preset) 开发。

**技术栈**: C++20 · CMake · Raylib 6.0 · Doxygen

## 基础类型详解 (`include/bruecken/common.h`)

所有自己写的代码都在 `namespace bruecken` 下。`common.h` 是全局基础，其他模块都要用它。

### 1. Position — 坐标

```cpp
struct Position {
    int x = 0;  // 列 (从左到右)
    int y = 0;  // 行 (从上到下)
};
```

坐标原点 `(0,0)` 在棋盘**左上角**，x 向右增大，y 向下增大。

### 2. Direction — 方向 / 棋盘边

```cpp
enum class Direction {
    kTop,     // 上边
    kBottom,  // 下边
    kLeft,    // 左边
    kRight,   // 右边
    kNone     // 无 / 重叠区
};
```

棋盘区域归属靠这个判断：
- 玩家 1 的边界区域：上边 + 下边
- 玩家 2 的边界区域：左边 + 右边
- 四个角是重叠区 (`kNone`)，双方都不能放棋子

```
     玩家1 上边
   ┌──────────┐
   │╲        ╱│
玩 │  ╲    ╱  │ 玩
家 │    ╳    │ 家
2  │  ╱    ╲  │ 2
左 │╱        ╲│ 右
   └──────────┘
     玩家1 下边
```

`opposite(Direction)` 返回对边：上↔下、左↔右。

### 3. Peg — 棋子

```cpp
struct Peg {
    Position pos;
    int player_id;  // 0 = 玩家1, 1 = 玩家2
};
```

棋子包含位置和归属。`player_id` 和 `preset::Move::get_id()` 对应。

### 4. Bridge — 桥梁

```cpp
struct Bridge {
    Position from;  // 起点
    Position to;    // 终点
    int player_id;  // 所属玩家
};
```

当两个同色棋子在「骑士跳」距离时会自动生成桥梁。

**骑士跳 (Rösselsprung)**: 横 1 纵 2 或横 2 纵 1（国际象棋马的走法）

```
  · · × · × · ·
  · × · · · × ·
  · · · ○ · · ·    ○ = 当前棋子
  · × · · · × ·    × = 可桥接位置
  · · × · × · ·
```

### 5. GamePhase — 游戏阶段

```cpp
enum class GamePhase {
    kNotStarted,  // 未开始
    kInProgress,  // 进行中
    kFinished,    // 有人赢了
    kDraw         // 平局
};
```

### 6. 常量一览

| 常量 | 值 | 含义 |
|------|-----|------|
| `kBoardMinSize` | 5 | 棋盘最小边长 |
| `kBoardMaxSize` | 96 | 棋盘最大边长 |

## 预设库关键类 (`lib/preset/`)

### preset::Player（最重要）

```cpp
class Player {
public:
    virtual std::string_view get_name() = 0;
    virtual void init(const BoardConfig& config, int player_id) = 0;
    virtual std::optional<Move> request() = 0;
    virtual void update(Move opponent_move) = 0;
};
```

所有玩家类型都继承这个。关键约定：
- `init()` 只能调一次，在其他函数之前
- `request()` 只在轮到自己时调一次，返回 `nullopt` 表示还没选好走法
- `update()` 在对手走棋后调，要验证对手走法是否合法，不合法就抛异常

### preset::Move

走法：只有坐标 `(x, y)` 和玩家 id。棋盘逻辑不在 Move 里，在 Board 里。

### preset::PlayerGuiAccess

```cpp
class PlayerGuiAccess {
public:
    virtual std::optional<Move> request_move_from_current_human_player() = 0;
};
```

GUI 必须继承这个接口。HumanPlayer 通过这个接口从 GUI 获取鼠标点击。

### preset::ArgumentParser

在 main 里构造，自动解析命令行参数。通过 `get_settings()` 拿到所有配置。
